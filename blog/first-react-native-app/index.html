ก้าวสู่การพัฒนาอีกขั้น จาก Web สู่ Mobile Application

27 มกราคม 2021

บริษัทของเรารับงานด้านการพัฒนา Website, Web Application ต่างๆบนระบบ Online โดยมุ่งเน้นไปที่ ภาคธุรกิจอุตสาหกรรม และทางด้านการศึกษาเป็นหลัก ทำให้ระบบส่วนใหญ่จะเป็น Website ที่เกี่ยวข้องกับการศึกษาหรือโปรแกรมงานด้านหลังบ้านที่ช่วยให้ ธุรกิจต่างๆทำงานได้ดี ง่ายดายมากยิ่งขึ้น

แต่ด้วยยุคสมัยที่เปลี่ยนไป มือถือ กลายเป็นสิ่งจำเป็นสำหรับคนทั่วไป สำหรับทุกธุรกิจและองค์กร ทางบริษัทเอง ก็เห็นถึงความสำคัญของการทำงานในรูปแบบ Mobile Application ไปไม่น้อยกว่ากัน แม้ตัว Web Applciation จะสามารถ เปิดได้บน Device หลากหลายขนาดแล้วก็ตาม แต่การทำงานก็ยังต้องอ้างอิง ความสามารถต่างๆจากทาง Browser เป็นหลักอยู่ดี

ทางบริษัทจึงได้ เริ่มต้นให้นักพัฒนาในทีม ได้เริ่มต้นเรียนรู้ เทคโนโลยีใหม่ๆ เพื่อให้รองรับกับการทำงานในรูปแบบของ Mobile Applciation มากยิ่งขึ้น แต่ทีมงานเอง ก็รู้สึกว่า Mobile จะว่าไปก็ไม่ได้นับว่าใหม่เลยซะทีเดียวนะครับ เพราะความที่เทคโนโลยีในตอนนี้ มีทางออก และทางเลือก ให้กับปัญหาทุกปัญหาของเราอยู่แล้ว เราได้เลือกว่าจะใช้ระหว่าง 2 เทคโนโลยี ในตอนนี้อย่าง Native Application หรือจะเป็น Hybrid Application ซึ่งเราลองมาดูวิธีเลือกของเรากันก่อนครับ

Native Application คืออะไร?
คือการเขียน Code ที่ทำงานเข้ากับ CPU ของหน่วยประมวลผลนั้นๆ โดยตรงได้เลย แบบที่ไม่จำเป็นต้องผ่านตัวแปลงภาษา พวก Virtual Machine เนื่องจากการเขียน Code เเหล่านั้น จะใช้ SDK หรือชุด library สำหรับพัฒนาจากเหล่า OS นั้นๆโดยตรง เช่น ถ้าเป็น iOS ก็จะใช้ Objective C ส่วนทาง Android ก็จะใช้ Android SDK ซึ่งเป็นชุดการพัฒนาหลักๆของแต่ละ Platform นั้นเอง ข้อดีคือ ทำให้มีประสิทธิภาพในการทำงานสูงมาก สามารถทำงานได้ครบถ้วน ทุกรูปแบบที่สามารถทำงานได้ของ features ฝั่งนั้นๆ
แต่ก็มีข้อเสียอยู่บ้างเหมทอนกัน คือ ต้องใช้เวลานาน เพราะต้องใช้นักพัฒนาที่เฉพาะด้าน และมีค่าใช้จ่ายสูง และหากต้องหารทำงานให้ รองรับทั้ง 2 ระบบก็ยิ่งต้องเพิ่มกำลัง และต้นทุนมากขึ้นเป็น 2 เท่าอีกด้วย

Hybrid Application คืออะไร?
จริงๆมันมีชื่อเรียก อีกอย่างว่า Cross Platform หรือการเขียน App ด้วยภาษาใดภาษาหนึ่ง แต่สามารถนำเอาไปใช้งาน ร่วมกันกับ Platform อื่นๆได้ด้วย ใช่...หมายถึง ถ้าคุณเขียนมันขึ้นมา 1 ครั้ง มันจะสามารถ Run ได้ทั้งบน iOS, Andrroid หรือแม้แต่บน Windows ก็ยังทำได้เลยทีเดียว
โดยปรกติ จะอาศับ Framework หรือ SDK ที่ถูฏพัฒนาขึ้นมาเพื่อให้ทำงานได้ ตรงจุดเป้าหมายมากที่สุด จากหลายหลายภาษาที่แตกต่างกันออกไป เช่น Flutter จากภาษา Dart หรือ Xamarin จาก C# และแน่นอนสำหรับตอนนี้ React Native จาก JavaScript ที่เป็นที่นิยมมากๆในตอนนี้
ข้อดีของ Hybrid ก็คือ เราจะประหยัดงบประมาณ แรงงาน และเวลาได้มากกว่า เพราะเราสามารถ ใช้คนเขียนแค่ทีมเดียว แล้วพัฒนาออกมาครั้งเดียว แต่สามารถทำงานได้ครบหมดทุก Platform และแม้ประสิทธิภาพ จะออกมาได้ไม่เต็มที่เท่ากับการทำงานของ Native แต่ก็มีส่วนเสริมมากมายที่ทำได้ใกล้เคียง หรือเรียกได้ว่า ครบครันมากพอสำหรับ Applciation ทั่วๆไป

และเพราะว่าทีมงานของเรา มี Base พื้นฐานมาจาก Online ซึ่งภาษาที่ถนัดก็คงเป็นในกลุ่มของการทำ Website และ 1 ในนั้นก็คือ JavaScript นั้นทำให้ เราเลือกที่จะพัฒนางานกันต่อไป ด้วยความสามารถของฝั่ง Hybrid Application ซึ่งแม้จะต้องเรียนรู้ Framework ใหม่ แต่ก็ไม่ได้เริ่มจาก 0 เพราะการที่เราเริ่มจาก Tool หรือตัวเครื่องมือ แปรว่าทักษะพื้นฐานที่เรามีอยู่นั้น สามารถนำมาใช้ในการพัฒนาเป็นต้นแบบ ก่อนก้าวเข้าสู่โลกของ Mobile Application กันได้อยู่แล้วนั้นเอง
